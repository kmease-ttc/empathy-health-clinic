# Audit Log & Observability Service

## Purpose
Provide traceability and system-wide visibility.

## What Is Logged
- Crawl runs
- Performance checks
- Content generation
- QA decisions
- Publishing actions

## Observability Features
- Run history
- Change diffs
- Job outcomes
- Failure alerts
- API quota warnings

## Alerts
- Crawl failures
- Indexing issues
- Traffic drops
- Service outages

## Usage Rule
All recommendations must be traceable to logged inputs.
